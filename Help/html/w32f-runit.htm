Run Win32Forth

You have two options to run Win32Forth : Console and IDE (Integrated Development Environment). The first one is the classical Forth console, an interactive commandline interface, the second one is a Windows interface where most commands can be made using mouse.

Console and IDE, however, work together : if you ask for editing a file from within the Console, the IDE, if not present, will be launched and open the file ; conversely, after editing, you can compile the file in the Console from within the IDE. So you get the best of both interfaces : ease of use of Windows plus interactive testing.

From either the Console or the IDE you can launch the Help too.

To switch from one of Console, IDE and Help to another one, just click in the title bar.

Win32For console

Run WIN32FOR.EXE.

You can edit, print or load a file using the Win32Forth File menu. With the edit menu, you can paste code to the Console to run it. The Display menu allows you to easily display various types of information. The Tools menu offers some useful facilities. The Macros menu is for recording and playing macros.

For beginners, we recommand that you type the tutorials' examples in the Console. You will get immediate feedback.

IDE

Run WIN32FORTHIDE.EXE.

For a basic use, you can think of the IDE as the current editor associated to the Console.

But the IDE, Integrated Development Environment, is much more. In addition to an Editor, it includes a project manager and a Form Designer without mentionning some extra goodies. The Project Manager helps you in the burden of managing many files in a project. The Form Designer makes Win32Forth somewhat "Visual" in the sense that it allows to interactively design forms, dialogs, etc. and associate code with them.

You will learn more about the IDE in the corresponding Help chapter.
