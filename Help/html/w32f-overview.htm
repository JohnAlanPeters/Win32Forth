Overview, Philosophy

Overview

Welcome to Win32Forth, a Windows implementation of the Forth programming language.

Win32Forth is ANS Forth standard compatible. As you can guess from its name, it is designed to develop 32bits Windows applications. It was initially written by Andrew McKewan, Tom Zimmer, Robert Smith and Jim Schneider during 1994 and 1995. Win32Forth is a FAT (as in BIG) system, adhering to the philosophy that if some tools are good, many tools must be better. It includes an interactive console and an integrated extensible source level debugger. Editing of source text is facilitated by the included WinEd and SciEdit file editors, which allows easy exploration of the many mega-bytes of source code that makes up Win32Forth.

Philosophy

Forth is not inherently a complex language, but when it is integrated into a Windows environment, some complexity creeps in. I have never been known as a simplist (one who seeks simplicity), but I do believe programs should be as simple as possible as long as they are not so simple that you can't easily get your job done. Basically, when faced with Windows, you can either say: "It is too complicated, I won't use it", or you can say: "it is a complex environment, I will attempt to master it and bring some order to the complexity". Both of these statements acknowledge that Windows is a big ugly creature, but the second statement also acknowledges that there may be some value in Windows that can be extracted and used to personal advantage. This is the approach I take, mostly because "it's my job", but also because some of the things Windows does are indeed interesting. If you, like me, find something interesting in Windows, then follow along. I can't tell you there won't be any pain or pitfalls associated with this little journey, but there will be discovery, excitement and lots and LOTS of exploration.
