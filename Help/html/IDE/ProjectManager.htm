Project Manager


The Project Manager helps managing the files associated with a Forth project. It allows files associated with an application to be easily transferred to other disk media as a standard zip file or by simply copying the files.

You may open a project from the IDE with the Project Manager Toolbar or menu option (see lower). You can also open Project Manager with a previously saved project (.fpj) file by double clicking it if the file association has been set up.

Project Manager tab

Click on the Project Manager tab. The following is a screenshot of an opened project.



The name of the opened project is displayed at the root of the project tree.

Files for an application are divided into the six categories shown :

  • Modules - Forth .f files
  • Forms - Form files, *.frm
  • DLLs - any Dlls needed
  • Resources - bitmap or icon files
  • Docs - .htm and .txt files.
  • Auxiliary - any other files

If any category contain files then clicking on that category name will expand the item and show the files. Clicking on a selected file will open it in the editor for editing or browsing.

The path name for a selected file is not shown in the treeview, however the full pathname is stored in the project manager. Note too that even if more than one file references or includes the same file only one copy is saved in the tree.

Project Toolbar & Menu

The Project toolbar and menu allow access to other Project Manager functions. For the toolbar hovering the mouse over a button will pop up a tooltip indicating the button's function.

The menu options are aas follows :
  • New - start a new project.
  • Close - close the current project.
  • Open - open a previously saved project. The name of the opened project is displayed in the statusbar.
  • Save - save current project. A project is saved as a standard text file.
  • Save As - save a copy of the project.
  • Rename - rename current project i.e the root name.
  • Build - build project. The build file would be the main file of the project. Scans the build file for needs, include and fload statements and add specified files. Recurses found files. Note that the build file is highlighted with a red icon in the project tree.
  • ReBuild - Same as build except any files currently in the project are deleted first.
  • Set search path for build - set the path to search for files with option to save to disk.
  • New Module - creates a blank .f file, adds it to the module list and opens the file in the editor ready for editing.
  • Navigator - Project Navigator : tracks code in a project.
  • Add files to project - opens a dialog allowing single or multiple file selection.
  • Delete from project - deletes highlighted file.
  • Add open Forms - Add any saved open forms to the forms list.

Compiling a project

If the option "Always compile main Project file" (from the Options Menu) is checked then the opened project will be compiled when "Compile" is selected from the menu or toolbar. If not checked then if the file in the active window is a source file it will be compiled.
